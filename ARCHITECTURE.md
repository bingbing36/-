## 🔄 数据流处理流程

### 1. 查询处理流程

```
用户输入 → 消息预处理 → 意图识别 → 检索策略选择
                              ↓
                    ┌─────────┴─────────┐
                    ↓                   ↓
            本地知识库检索        网络信息搜索
            (Elasticsearch)       (Tavily API)
                    ↓                   ↓
                    └─────────┬─────────┘
                              ↓
                    信息融合 + 来源标注
                              ↓
                    LLM 生成回答
                              ↓
                    结果后处理 + 格式化
                              ↓
                    用户界面展示
```

### 2. 文档索引流程

```
原始文档 (TXT/PDF)
    ↓
文档加载和预处理
    ↓
分块处理 (Chunking)
    ↓
向量化 (Embedding)
    ↓
Elasticsearch 索引存储
    ↓
索引统计和验证
```

## 🏗️ 核心组件设计

### 1. 智能体框架 (Agent)

**职责**：对话管理和决策调度

```python
class Assistant(Agent):
    def __init__(self, name, llm, system_message, function_list, files, rag_cfg):
        # 初始化
        self.name = name                 # 助手名称
        self.llm = llm                  # LLM 配置
        self.system_message = system_message  # 系统提示词
        self.function_list = function_list  # 可用工具列表
        self.rag_cfg = rag_cfg          # RAG 检索配置
        
    def run(self, messages):
        # 处理一轮对话
        # 1. 分析用户意图
        # 2. 调用相关工具
        # 3. 生成回答
        # 返回: 生成器，逐步返回结果
```

### 2. RAG 检索模块

**职责**：从知识库中检索相关信息

```python
# 向量化和检索流程
documents → 分块 → 向量化 → 索引存储
                              ↓
                         查询向量化
                              ↓
                         相似度计算
                              ↓
                         Top-K 检索
                              ↓
                         返回相关文档
```

**职责**：扩展应用功能

**系统工具**：

## 🎯 设计原则

1. **模块化**：各组件独立，易于维护和扩展
2. **可配置**：尽量减少代码改动，通过配置调整
3. **可观测**：详细日志和监控指标
4. **高性能**：缓存、异步、并发优化
5. **安全性**：数据加密、访问控制、审计日志
6. **可扩展**：易于添加新工具、新模型、新数据源
